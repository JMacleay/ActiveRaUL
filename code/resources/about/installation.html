Installation

To follow the instructions below how to install the web service component of the ActiveRaUL system on your server, you need to first download the WAR file from our Google Code project.

Prerequisites

  • Java 5 or newer
  • Apache Ant
  • A Java Servlet Container that supports Java Servlet API 2.4 and Java Server Pages (JSP) 2.0, or newer.
  • Sesame 2 RDF Store installed

The ActiveRaUL system has been tested with Java version "1.7.0", Ant version "1.8.2", Tomcat 6 and Sesame 2.6.3.

Deploying the ActiveRaUL Web service

The easiest way to install the ActiveRaUL service on Tomcat is using the Tomcat Web Application Manager to upload the "raul.war" file. Otherwise, you can manually deploy the ActiveRaUL service on Tomcat according to the following steps.

  1. Build the ActiveRaUL service from source by typing "ant clean; ant dist" under the directory that contains the ActiveRaUL source code.
  2. Create a native repository with RDF Schema inferencing on Sesame server, and designate the newly create repository as "raul-default". (Please refer to the documents of Sesame server for how to create an RDF repository.)
  3. Extract the "raul.war" file to the web applications directory ([TOMCAT_HOME]/webapps/ by default).
  4. Restart your Tomcat server and the ActiveRaUL service should now be up and running.

Note that you can change the configuration of the RDF repository setting by editing "raul.properties" under the "/src/main/java" directory. Having done so, you should reload the ActiveRaUL service to activate the change.

Test the service

You can point a web browser to the location where you have installed Sesame (e.g. http://[MACHINE_NAME]:8080/raul/ if you have installed the ActiveRaUL service under Tomcat) to test your installation. The browser should now display a demo web site of the ActiveRaUL system.
