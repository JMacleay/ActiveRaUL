RaUL:Button

The Button form control is used for triggering or non-triggering actions. Beyond the common attributes inherited from the Widget superclass the Button class defines the following properties, checked, command, disabled and group. The mapping to HTML creates either a normal push button (ie. an input field of type button) or a submit button in case the command property is set to "submit". A submit button is used to trigger the action defined in the form element. The Button class has two subclasses in the RaUL vocabulary, a Radiobutton and a Checkbox. They are mapped to input controls of type radio or of type checkbox, respectively. To group buttons together and determine the selected values, the group property of the Button class can be used.
For the data binding a similar restriction to the one of Listboxes applies. Any Group of Buttons, ie. either Radiobuttons or Checkboxes, which values are not literals, must bind to an RDF collection with the same number of node elements as there are Buttons in the respective group. After submission of the Button control element the checked property is set for all selected Checkboxes or Radiobuttons.
The listing below shows the submit button from our ProductAdd example. The raul:command property is set to the submitDataWrap() method in the ActiveRaUL Client-side library.

<rdf:RDF
  xmlns:rdf="http://www.w3.org/1999/02/22-rdf-syntax-ns#"	
  xmlns:raul="http://purl.org/NET/raul#"

<rdf:Description rdf:about="http://w3c.org.au/raul/service/public/forms/productAdd#submit">
  <rdf:type rdf:resource="http://purl.org/NET/raul#Button"></rdf:type>
  <raul:command>submitDataWrap()</raul:command>	
  <raul:id>Submit</raul:id>
  <raul:name>Submit</raul:name>
  <raul:value>Submit</raul:value>
</rdf:Description>
