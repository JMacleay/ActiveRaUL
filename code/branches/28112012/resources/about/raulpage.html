RaUL:Page

There are two ways to use a RaUL Web form in a browser. Depending on the existence of the Page class in the RaUL form model, either an entire HTML page, or a widget that is embeded in a <div> tag is rendered. If an entire page is rendered, the Web developer does not need to change anything for the client rendering. ActiveRaUL will include the reference to the required JavaScript libraries in the rendering of the HTML page. However, following this approach, the Web developer is limited in the design of the page to the changes possibly through CSS.

The Page class constitutes the main container of a RaUL document. In case of the mapping to XHTML a Page instance maps to a element. The Page class defines some common attributes such as class, id and title. The class and id attribute can be used to bind a CSS style identifier to specify the display size of a form control. A Page can contain one or many WidgetContainers. Below an example of how to use the Page class.

<rdf:RDF	
	xmlns:owl="http://www.w3.org/2002/07/owl#"
	xmlns:rdf="http://www.w3.org/1999/02/22-rdf-syntax-ns#"	
	xmlns:raul="http://purl.org/NET/raul#">

<rdf:Description rdf:about="http://w3c.org.au/raul/service/public/forms/productAdd">
	<rdf:type rdf:resource="http://purl.org/NET/raul#Page"></rdf:type>
	<raul:id>content</raul:id>
	<raul:list>http://w3c.org.au/raul/service/public/forms/productAdd#widgetContainerList</raul:list>
</rdf:Description>

<rdf:Description rdf:about="http://w3c.org.au/raul/service/public/forms/productAdd#widgetContainerList">
	<rdf:type rdf:resource="http://www.w3.org/1999/02/22-rdf-syntax-ns#Seq"></rdf:type>
	<rdf:_1 rdf:resource="http://w3c.org.au/raul/service/public/forms/productAdd#container"></rdf:_1>
</rdf:Description>
