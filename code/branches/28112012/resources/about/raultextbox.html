RaUL:Textbox

Textbox form controls allow free-form data entry and map to an input box of type text in the XHTML rendering. Properties of the Textbox are, disabled, hidden, isPassword, maxlength, row and size. These properties are straightforwardly mapped to their equivalents in the XHTML model.

The actual binding of the data submitted through the Textbox to concepts in an RDF ontology is realised via reification within RDF. Reification in RDF describes the ability to treat a statement as a resource, and hence to make assertions about that statement. Fig. 4 shows, for example, how the Product Name Textbox Widget references the RDF triple representing the corresponding underlying data element (i.e., <http://...><gr:name> <''>). The rdf:subject triple references the URI assigned by the ActiveRaUL service for the instance graph, the rdf:predicate triple is a reference to the URI of a standard Web ontology property, in this case the GoodRelations name property, and the rdf:object triple is a reference to the value that can be edited by the Textbox form control. Empty rdf:object fields serve as place-holders and are filled at runtime by the ActiveRaUL client-side JavaScript library with the user input.

<rdf:RDF
  xmlns:rdf="http://www.w3.org/1999/02/22-rdf-syntax-ns#"	
  xmlns:raul="http://purl.org/NET/raul#"
  xmlns:gr="http://purl.org/goodrelations/v1#">

<rdf:Description rdf:about="http://w3c.org.au/raul/service/public/forms/productAdd#product_name">
  <rdf:type rdf:resource="http://purl.org/NET/raul#Textbox"></rdf:type>
  <raul:label rdf:datatype="http://www.w3.org/2001/XMLSchema#string">Product Name:</raul:label>
  <raul:id>product_name</raul:id>
  <raul:name>product_name</raul:name>
  <raul:value>http://w3c.org.au/raul/service/public/forms/productAdd#value_product_name</raul:value>
</rdf:Description>

<rdf:Description rdf:about="http://w3c.org.au/raul/service/public/forms/productAdd#value_product_name">
  <rdf:subject></rdf:subject>
  <rdf:predicate>gr:name</rdf:predicate>
  <rdf:object></rdf:object>
</rdf:Description>
